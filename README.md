# OS

Laboratory Work on Operating Systems

Репозиторий содержит лабораторные работы по дисциплине **«Операционные системы»**.  
В работах рассматриваются базовые механизмы ОС: командные оболочки, планирование процессов, виртуальная память и многопоточность.

## Лабораторные работы

### Лабораторная работа №1 — Разработка оболочки

Реализация простой командной оболочки `myshell` для UNIX-подобной системы.

Основные возможности:
- выполнение внутренних команд: `cd`, `clr`, `dir`, `environ`, `echo`, `help`, `pause`, `quit`;
- запуск внешних программ через `fork` и `exec`;
- поддержка batch-режима;
- перенаправление ввода и вывода;
- фоновое выполнение процессов;
- отображение текущего каталога в приглашении командной строки.

### Лабораторная работа №2 — Алгоритмы планирования

Реализация алгоритмов планирования процессов на языке C.

Рассматриваемые алгоритмы:
- FCFS — First Come, First Served;
- SJF — Shortest Job First;
- приоритетное планирование;
- Round Robin;
- Round Robin с приоритетами.

Дополнительно может выполняться подсчёт среднего времени ожидания, оборотного времени и времени отклика.

### Лабораторная работа №3 — Менеджер виртуальной памяти

Проект посвящён моделированию преобразования логических адресов в физические.

Реализуются:
- таблица страниц;
- TLB;
- обработка страничных ошибок;
- подкачка страниц из `BACKING_STORE.bin`;
- расчёт статистики: частота ошибок страниц и частота попаданий в TLB.

Дополнительно может быть реализовано замещение страниц при ограниченном количестве физических фреймов.

### Лабораторная работа №4 — Проектирование пула потоков

Реализация пула потоков с использованием POSIX Threads.

Основные элементы проекта:
- создание рабочих потоков;
- очередь задач;
- синхронизация с помощью mutex и semaphore;
- добавление задач через `pool_submit`;
- корректное завершение работы пула через `pool_shutdown`.

## Используемые технологии

- C
- UNIX/Linux
- POSIX API
- PThreads
- Makefile

## Цель репозитория

Закрепить практические навыки работы с механизмами операционных систем:
- процессами;
- потоками;
- планированием;
- памятью;
- синхронизацией;
- системными вызовами UNIX.
